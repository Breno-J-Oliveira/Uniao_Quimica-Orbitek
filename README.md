## O que é o projeto União Química?

A **União Química**, empresa de destaque no setor farmacêutico, solicitou à equipa **Orbitek** o desenvolvimento de um sistema digital para otimizar os seus processos internos. O projeto foi estruturado em **3 sprints**, cobrindo o ciclo completo de desenvolvimento: prototipagem e interfaces (Sprint 1), modelagem de base de dados MySQL (Sprint 2), e integração PHP com operações CRUD (Sprint 3).

O sistema contempla **5 páginas principais** — Login, Cadastro de Administrador, Cadastro de Fornecedores, Cadastro de Produtos e Listagem de Produtos — com design responsivo prototipado no Figma (versões computador e celular) e deploy em produção no InfinityFree.

**Do protótipo no Figma ao deploy em produção com CRUD funcional.**

---

- **Slides profissionais no Canva** — apresentação do projeto

- **Modelagem de dados corporativa** — 3 modelos (conceitual, lógico, físico) documentados
- **Full stack** — HTML/CSS (front-end) + PHP (back-end) + MySQL (base de dados)

- **Deploy em produção** — site acessível online via InfinityFree

## Como funciona? — Percurso das 3 Sprints

### 🔵 Sprint 1 — Prototipagem e Interfaces

```
1. Protótipo no Figma — versão Computador + Celular
   5 páginas desenhadas (Login, Admin, Fornecedores, Produtos, Listagem)
   Design system com paleta de cores corporativa
       ↓
2. Desenvolvimento HTML/CSS
   5 páginas codificadas com Font Awesome
   Estrutura semântica e responsiva
       ↓
3. Apresentação e validação
   Slides no Canva, escopo definido
```

### 🔴 Sprint 2 — Modelagem de Base de Dados

```
4. Dicionário de Dados
   Definição de entidades, atributos e tipos
       ↓
5. Modelo Conceitual (DER)
   Diagrama Entidade-Relacionamento
   Entidades: usuários, fornecedores, produtos
       ↓
6. Modelo Lógico
   Tabelas, colunas, tipos de dados, chaves
   Normalização e integridade referencial
       ↓
7. Modelo Físico (MySQL Workbench)
   Script SQL, índices, constraints
   Pronto para implementação
```

### 🟠 Sprint 3 — Integração PHP e CRUD

```
8. Conexão PHP ↔ MySQL
   Integração do front-end com a base de dados
       ↓
9. Operações CRUD
   CREATE, READ, UPDATE, DELETE
       ↓
10. Deploy em Produção
    Hospedagem no InfinityFree
    Site funcional online
```

# 🟩 SPRINT 2

A **Sprint 2** tem como foco a modelagem dos dados do sistema, que envolve quatro etapas fundamentais: o **dicionário de dados**, o **modelo conceitual**, o **modelo lógico** e o **modelo físico**.

---

## 🗄️ MySQL Workbench

Para dar suporte às funcionalidades do sistema, foi desenvolvido um banco de dados relacional no **MySQL Workbench**, garantindo a persistência e a organização dos dados de administradores, fornecedores e produtos.

O modelo foi pensado para refletir diretamente as telas do sistema:

- **Usuários/Administradores** — controlo de acesso ao sistema
- **Fornecedores** — cadastro de parceiros e contactos
- **Produtos** — informações detalhadas para listagem e consulta

---

## 📐 Modelo Conceitual

O modelo conceitual de banco de dados é uma representação abstrata e de alto nível dos dados de um negócio, focada em entidades, atributos e relacionamentos, sem detalhes técnicos de implementação.

Ele usa ferramentas como o **Diagrama Entidade-Relacionamento (DER)** para traduzir requisitos em visuais claros, facilitando a comunicação entre equipas e servindo de base para os modelos lógico e físico.

<img width="400" height="400" alt="Modelo Conceitual" src="https://github.com/user-attachments/assets/3b7cbfb5-3a7e-4c8e-baf2-a914b2f25f34" />

## 📊 Modelo Lógico

O modelo lógico de banco de dados é uma representação detalhada e independente de SGBD específico, derivada do modelo conceitual, que converte entidades em tabelas, atributos em colunas com tipos de dados e relacionamentos em chaves primárias e estrangeiras.

Ele incorpora restrições de integridade, regras de negócio e normalização para garantir consistência e eficiência, facilitando a transição para a implementação física.

<img width="400" height="400" alt="Modelo Lógico" src="https://github.com/user-attachments/assets/e3da43f1-bda8-431a-bf6c-9f9b6b0a2c1a" />

<p align="center"><em>Nosso modelo Lógico do Banco de Dados</em></p>

---

## 💾 Modelo Físico

O modelo físico de banco de dados é a representação final e dependente do SGBD específico, derivada do modelo lógico, que detalha a implementação real com tabelas, colunas, tipos de dados nativos, índices, restrições, views e procedimentos armazenados.

Ele considera otimizações de desempenho, como particionamento de tabelas, armazenamento físico e convenções de nomenclatura do banco escolhido (ex.: MySQL, Oracle), permitindo a criação direta do esquema no sistema.

# 🟧 SPRINT 3

A **Sprint 3** tem como objetivo principal integrar o **PHP** ao banco de dados desenvolvido na Sprint 2, estabelecendo a conexão entre o site e a base de dados. Essa etapa é essencial para possibilitar a manipulação dinâmica das informações pelo utilizador.

---

## 🐘 PHP

No âmbito do projeto da Orbitek, o **PHP** é empregado para integrar os modelos de dados — conceitual, lógico e físico — à interface web, possibilitando o desenvolvimento de aplicações dinâmicas que acessam e manipulam bancos de dados de maneira eficiente e escalável.

PHP é uma linguagem de programação usada para criar sites dinâmicos, que interagem com utilizadores e bancos de dados. Ela é executada no servidor e permite desenvolver sistemas web de forma simples e eficiente.

---

## 🗃️ CRUD

No âmbito do projeto da Orbitek, as operações **CRUD** são aplicadas para gerir eficientemente os dados nos modelos de banco de dados desenvolvidos, garantindo a manipulação completa de informações em aplicações web.

CRUD é um conjunto de operações básicas realizadas em bancos de dados: **Criar, Ler, Atualizar e Excluir** dados. Essas funcionalidades facilitam o gerenciamento de informações em aplicações web.
