## O que é o projeto União Química?

A **União Química**, empresa de destaque no setor farmacêutico, solicitou à equipa **Orbitek** o desenvolvimento de um sistema digital para otimizar os seus processos internos de cadastro. O projeto foi estruturado em **3 sprints**, cobrindo o ciclo completo de desenvolvimento: prototipagem e interfaces (Sprint 1), modelagem de base de dados MySQL (Sprint 2), e integração PHP com operações CRUD (Sprint 3).

O sistema contempla **5 páginas principais** — Login, Cadastro de Administrador, Cadastro de Fornecedores, Cadastro de Produtos e Listagem de Produtos — com design responsivo prototipado no Figma (versões computador e celular) e deploy em produção no InfinityFree.

**Do protótipo no Figma ao deploy em produção com CRUD funcional.**

## Como funciona? — Percurso das 3 Sprints

### 🔵 Sprint 1 — Prototipagem e Interfaces

```
1. Protótipo no Figma — versão Computador + Celular
   5 páginas desenhadas (Login, Admin, Fornecedores, Produtos, Listagem)
   Design system com paleta de cores corporativa
       ↓
2. Desenvolvimento HTML/CSS
   5 páginas codificadas com Font Awesome
   Estrutura semântica e responsiva
       ↓
3. Apresentação e validação
   Slides no Canva, escopo definido
```

**Objectivo:** criar o design visual completo e as interfaces estáticas. Validar com o cliente.

### 🔴 Sprint 2 — Modelagem de Base de Dados

```
4. Dicionário de Dados
   Definição de entidades, atributos e tipos
       ↓
5. Modelo Conceitual (DER)
   Diagrama Entidade-Relacionamento
   Entidades: usuários, fornecedores, produtos
       ↓
6. Modelo Lógico
   Tabelas, colunas, tipos de dados, chaves
   Normalização e integridade referencial
       ↓
7. Modelo Físico (MySQL Workbench)
   Script SQL, índices, constraints
   Pronto para implementação
```

**Objectivo:** modelar a base de dados relacional completa com 3 níveis de abstração.

### 🟠 Sprint 3 — Integração PHP e CRUD

```
8. Conexão PHP ↔ MySQL
   Integração do front-end com a base de dados
       ↓
9. Operações CRUD
   CREATE: cadastrar fornecedores e produtos
   READ: listar e consultar dados
   UPDATE: editar registos existentes
   DELETE: remover registos
       ↓
10. Deploy em Produção
    Hospedagem no InfinityFree
    Site funcional online
```

**Objectivo:** tornar o sistema dinâmico com PHP + MySQL e publicar online.

## 🗄️ Modelagem de Base de Dados

### Sprint 2 — 3 Níveis de Abstração

| Nível | Descrição | Ferramenta |
|-------|-----------|------------|
| **Modelo Conceitual** | Diagrama Entidade-Relacionamento (DER) — entidades, atributos, relacionamentos | MySQL Workbench |
| **Modelo Lógico** | Tabelas com colunas, tipos de dados, chaves primárias e estrangeiras, normalização | MySQL Workbench |
| **Modelo Físico** | Script SQL pronto para execução, índices, constraints, engine InnoDB | MySQL Workbench |

### Entidades

| Entidade | Descrição |
|----------|-----------|
| **usuarios / administradores** | Controlo de acesso ao sistema (login, senha criptografada) |
| **fornecedores** | Cadastro de parceiros com dados de contacto |
| **produtos** | Informações detalhadas vinculadas a fornecedor (chave estrangeira) |

### Integridade e Segurança

- **Integridade referencial:** cada produto vinculado a um fornecedor via FK
- **Senhas criptografadas:** armazenamento seguro com hash
- **Escalabilidade:** estrutura preparada para futuras expansões (histórico de vendas, relatórios)
